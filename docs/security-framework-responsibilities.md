# 本项目中的安全分工

## 框架职责

### Spring Boot
- 提供全局异常处理、拦截器、过滤器和配置管理能力。
- 适合做安全基建能力：统一返回格式、参数预处理、敏感日志过滤。

### Spring Security
- 负责认证与授权主干：登录态校验、接口访问控制、方法级权限控制。
- 可覆盖未授权访问、越权访问控制、部分会话安全策略。
- JWT 校验链路通常挂在其过滤器链中。

### Spring Validation（jakarta validation）
- 负责入参合法性校验：非空、长度、格式、范围。
- 可拦截大量恶意输入和脏数据。
- 注意：参数校验不等于 SQL 注入防护。

### MyBatis
- 负责数据库访问层的 SQL 安全细节。
- 使用 `#{}` 预编译参数可防大部分 SQL 注入。
- `${}` 动态字符串拼接风险高，必须用严格白名单控制。

### Spring Data Redis
- 支持 token 撤销、黑名单、刷新防重放、限流计数等能力。
- 可覆盖登出失效、强制下线、refresh token 轮换等场景。

### Spring Cloud Gateway（mall-gateway）
- 提供网关侧统一安全能力：前置鉴权、限流、黑名单、CORS 策略、统一审计入口。
- 可在业务服务之前拦截大量异常流量（如爆破、接口刷流量）。

### Log4j2 / 日志体系
- 支持安全审计与追踪：登录失败、刷新失败、异常访问行为。
- 日志本身不直接防攻击，但对检测与事后追溯非常关键。

### Docker / 部署层
- 提供运行时隔离与环境一致性。
- 不直接防业务攻击，但可配合最小权限、网络隔离、镜像漏洞扫描提升整体安全。

## 仅靠框架无法自动解决的风险
- 业务级越权（例如用户查看他人订单）需要在业务逻辑中明确校验。
- 密钥管理、证书管理、漏洞修复流程依赖工程与运维规范。
- 风控策略（设备指纹、异地登录、行为分析）需要产品与系统层面的专项设计。
